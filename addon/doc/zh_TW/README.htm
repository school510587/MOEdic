1. 簡介

　　「國字快查」讓視覺障礙用戶（以下簡稱「視障者」）可以相當迅速地向「中華民國教育部 (Ministry of Education, R.O.C.) 《重編國語辭典修訂本》」（以下簡稱「教育部重編國語辭典修訂本」）查詢想要的資料。針對視障者使用螢幕閱讀軟體 (Screen Reader) 的鍵盤操作來設計，把網頁上的查詢步驟簡化，並將查詢結果的文字資料內容呈現於頁面，便於視障者檢索。
　　本程式始於 NVDA 台灣郵遞論壇，由 嘯傲俠羽 提出需求，Coscell Kao 著手開發「國字快查」的初版，之後由 Sponge Jhan 改寫，完成本次的釋出版本。目前的「國字快查」已經達成下列目標：
  • 將用戶需要的操作與閱讀量減至最低。
  • 能夠穩定抓取「教育部重編國語辭典修訂本」對每次查詢所回應的全部資料。
  • 多元化的操作方式，用戶不但能透過 NVDA 及導盲鼠、晴光等螢幕閱讀軟體，也可以用視覺來操作「國字快查」。

1.1. 緣起

　　最初，現任台中啟明教師的嘯傲俠羽因其國小老師的職責，指導視障學童上網查詢「教育部重編國語辭典修訂本」。嘯傲俠羽本身也是視覺障礙，他在頻繁的操作中，仍覺得視障者操作網頁相對明眼人而言較不順暢。再者，教育上必須培養學童獨立操作、學習之能力，但是年紀太小的學童並無法馬上學習操作任何字典的網頁介面。嘯傲俠羽遂提出簡化「教育部重編國語辭典修訂本」操作程序的需求，於是 Coscell Kao 應允並著手開發「國字快查」。不過該初版「國字快查」仍存在錯誤與不穩定。並且，考慮前述教育之目的，「國字快查」也應該顯示訊息讓明眼協助者看見視障用戶的操作狀況，適時給予指導。之後由 Sponge Jhan 接手改寫，完成本次的釋出版本。「國字快查」的開發與改進，除了歸功於提案與開發者之外，也要感謝開發期間協助開發者們測試的所有人員。

1.2. 許可證

　　「國字快查」為自由軟體，在 GNU 公共授權條款 (GNU General Public License, GPL) 下發佈，詳見 LICENSE.txt. 另外，由於「國字快查」顯示的資料內容直接取自中華民國教育部的網站，在使用這些資料之前請詳閱「教育部國語辭典公眾授權網」之規定。

2. 使用說明

  • ◎　請用 Win 加九宮格數字 2 鍵執行「國字快查」的附加元件，或執行「國字快查」的執行檔以啟動它。
  • ◎　按一下 Ctrl 加 Win 加 Q 檢查目前「國字快查」的狀態，快按兩下則顯示「簡易查詢結果」或繼續抓取資料。
  • ◎　用戶可在「簡易查詢結果」中翻頁檢視資料，若將該視窗正常關閉則結束查詢。
　　「國字快查」支援 NVDA 及導盲鼠、晴光等螢幕閱讀軟體，但需要在 Windows XP Service Pack 3 或以上版本的 Windows 系統中，配合 Internet Explorer 8 或以上版本的瀏覽器執行。

2.1. 安裝

　　在閱讀本段之前，我們假設用戶已經將含有此文件的檔案 MOEdic.zip 解壓縮，解開後除了此文件 (README.htm) 之外，有一個 MOEdic-EXE 資料夾跟一個 MOEdic-x.x.x.nvda-addon 檔案，其中 x.x.x 表示檔案版本的三個數字，如 1.2.3.

2.1.1. NVDA 用戶

　　NVDA 用戶請直接執行 MOEdic-x.x.x.nvda-addon 並依照 NVDA 指示完成附加元件的安裝。安裝完成之後，附加元件會以「國字快查」的名稱註冊於「管理附加元件」的清單中。

2.1.2. 其他螢幕閱讀軟體用戶

　　其他螢幕閱讀軟體用戶請進入 MOEdic-EXE 資料夾，並依據自己的環境挑選 MOEdic.exe 或 MOEdic-x64.exe 來使用，二個執行檔分別對應 32 位元及 64 位元之 Windows 作業系統。
　　注意：雖然 64 位元的使用環境下可以執行 32 位元的應用程式，但是這種方式執行的「國字快查」極不穩定。因此，如果用戶不清楚自己的使用環境為幾位元，請先嘗試直行 MOEdic-x64.exe, 成功執行則您的系統是 64 位元，遇到失敗（不是正確的 Win32 應用程式）則系統是 32 位元。

2.2. 操作

2.2.1. 啟動

2.2.1.1. NVDA 用戶

　　NVDA 用戶請按 Win 鍵加九宮格數字 2 來啟動「國字快查」。若成功啟動，附加元件會先試著從剪貼簿取得輸入，當剪貼簿資料不合輸入查詢詞格式時，才顯示對話框要求用戶輸入。然而，如果附加元件上次啟動的執行尚未結束，用戶會收到「已經在查詢了」的訊息。

2.2.1.2. 其他螢幕閱讀軟體用戶

　　其他螢幕閱讀軟體用戶請直接執行 MOEdic.exe 或 MOEdic-x64.exe 來啟動「國字快查」。成功啟動後即顯示輸入查詢詞的對話框，如果剪貼簿中有文字內容，會將其填入編輯區做為預設的查詢詞。

2.2.1.3. 輸入查詢詞之對話框

　　輸入查詢詞之對話框中除了接受用戶輸入的編輯區，還有 [OK] 與 [Cancel] 二個按鈕。輸入完畢可按 [OK] 開始查詢，或 [Cancel] 取消並關閉「國字快查」。對話框也允許某些快速鍵的操作，焦點在編輯區按 Enter 效果與點擊 [OK] 相同，按 Esc 或 Alt 加 F4 視同取消並關閉。
　　查詢詞必須符合格式才允許被送出。查詢詞格式為不超過 20 字元的含逗號中文字串或純英文單字（大小寫不拘）。如果格式錯誤，「國字快查」會要求用戶重新輸入。

2.2.2. 檢視查詢進度與結果

　　當「國字快查」在螢幕或點字顯示器呈現「開始查詢」訊息以後，用戶可以透過 Ctrl 加 Win 加 Q 與之互動。該快速鍵被按一次時，「國字快查」會在螢幕或點字顯示器上呈現目前抓取資料的進度跟狀態。如果狀態為「正在抓取資料」，則可快按兩次該快速鍵（按住 Ctrl 和 Win 鍵並快按兩下 Q 鍵）來要求「國字快查」暫停並顯示「簡易查詢結果」頁面，讓用戶查閱目前所抓到的資料。
　　「簡易查詢結果」是一個以 Internet Explorer (IE) 瀏覽器開啟的網頁。所有查詢到的資料會在該網頁上分好幾頁顯示，用戶可以依自己需求調整每頁顯示資料的多寡。當焦點停留在該網頁上時，如果資料還沒抓完，用戶可再快按兩次 Ctrl 加 Win 加 Q 讓「國字快查」關閉網頁並繼續抓取資料。關於「簡易查詢結果」網頁其他的操作細節請見下一小節。
　　如果用戶耐心等到資料全部抓完，沒有按兩下 Ctrl 加 Win 加 Q 暫停，「國字快查」將自己停止抓取資料，把含有「簡易查詢結果」網頁的瀏覽器視窗以最小化型態顯示在工具列，並發出提示音效提醒用戶。用戶從工具列將該瀏覽器視窗叫出來之後，必須多按一個（或數個）Tab 鍵把焦點停在網頁中，方可進行閱讀、換頁等操作。

2.2.3. 瀏覽查詢結果

　　「簡易查詢結果」頁面分為三大區塊：導航區、主要內容區與資訊區。

2.2.3.1. 頁面配置

　　用戶可在導航區裡設置每頁顯示多少資料與目前顯示第幾頁。當用戶點擊呈現頁數的連結，網頁會顯示對話框讓用戶輸入想要前往的頁數，用戶也可以透過旁邊的「上一頁」、「下一頁」按鈕來逐頁瀏覽。用戶可在網頁最上方的下拉方塊選擇每頁顯示多少資料。當選擇完畢，網頁會重設頁數在第一頁。
　　主要內容區列出從「教育部重編國語辭典修訂本」所查詢到的資料，包括詞彙本身、注音、漢語拼音、釋義等。每筆資料都附有一個連結，指向「教育部重編國語辭典修訂本」的原始網頁。用戶可直接點擊該連結以針對某個詞彙做進一步的查詢。
　　資訊區包含一個回到頁首的連結，以及指向 NVDA 台灣郵遞論壇的連結。使用上遇到問題或者有新功能的需求都可以透過該論壇跟「國字快查」的開發者取得聯繫。

2.2.3.2. 鍵盤操作

　　無論焦點在頁面何處，用戶都可以按 Alt 加 P 回上一頁，以及 Alt 加 N 跳下一頁。換頁動作執行完成時，用戶的焦點會停在呈現頁數的連結上，讓用戶立即知道現在處於第幾頁。
　　NVDA 的用戶更可善用「單鍵瀏覽」功能，在頁面上用 K 鍵先閱讀每個辭彙，遇到有興趣者再以方向鍵上下操作閱讀注音、拼音、釋義、相似／相反詞等訊息。註：「單鍵瀏覽」的細節不在本說明範圍內，詳見 NVDA 之用戶指南。

2.2.4. 結束查詢

　　將查詢網頁用 Alt 加 F4 或點擊右上角的叉叉關閉視窗，「國字快查」即結束查詢，用戶可再啟動它以進行下次查詢。

3. 注意事項

　　在使用「國字快查」時，敬請用戶詳讀下列注意事項：
  1. 建議用戶將 IE 瀏覽器設為盡量以視窗代替分頁，以獲得較佳的使用經驗。
  2. 請勿同時執行「國字快查」附加元件與執行檔，也不可以同時執行多次執行檔，否則將只有第一次執行會成功註冊快速鍵，其他次執行皆失敗。
  3. 當「國字快查」正在抓取資料時，請暫停其他的 IE 瀏覽器操作，否則可能影響「國字快查」的執行。
  4. 有時候「國字快查」會自己暫停抓取資料，這種狀況會發生在已經抓取很多（數百）筆資料後。「國字快查」會採取如同前述抓完所有資料之行為，但用戶可以從工具列開啟「簡易查詢結果」網頁並快按兩次 Ctrl 加 Win 加 Q 讓「國字快查」繼續抓取剩下的資料。
  5. 在「簡易查詢結果」網頁中，「上一頁」與「下一頁」按鈕會因為沒有上（下）一頁而變成無法使用的狀態，然而 NVDA 似乎不會完全與按鈕狀態同步回報給用戶。因此當用戶發現按鈕不對 Enter 等鍵入有反應時，表示已無上（下）一頁可以切換。
  6. 請勿將「簡易查詢結果」網頁所在的瀏覽器視窗導向到其他網頁，如 Alt 加 Home 回到首頁等。
  7. 若 NVDA 用戶在執行「國字快查」附加元件中遭遇問題，可從 NVDA 的事件記錄中查看剛才「國字快查」執行的狀況，回報問題時將這些內容交給開發者將有助於解決用戶所遭遇的問題。
